## 1. Видео-плеер (`player.html`) — хороший, минимальные правки

### Что хорошо
- Чистый Tailwind + Inter font — выглядит профессионально
- Адаптивный 2/3 + 1/3 grid
- Sidebar с материалами sticky (не уезжает при скролле)
- 16:9 соотношение iframe через padding-bottom trick — правильно

### Что исправить
1. **Хардкод бот-ссылки** — `href="https://t.me/DavydovaAIBot"` захардкожен. Нужно передавать через Jinja2 из config: `{{ bot_url }}`.
2. **Нет навигации между уроками** — пользователь смотрит урок, заканчивает, нажимает «Вернуться в бот», потом снова выбирает следующий урок. Можно добавить кнопки «← Предыдущий» / «Следующий →».

## 6. Бот (`bot.py`) — критические проблемы

### 🔴 БАГ: Видео-ссылка не показывается в сообщении

В `process_lesson()` (строка 443–460):
```python
lesson_text = (  # ← переменная создаётся
    f"*{lesson.title}*\n\n"
    ...
    f"🔗 *Ссылка на видео:*\n"
    f"`{display_url}`\n\n"
)
...
await callback.message.edit_text(text, ...)  # ← используется `text`, не `lesson_text`!
```

Результат: пользователь видит только заголовок урока и описание. Ссылка на видео в теле сообщения не показывается. Кнопка «▶️ СМОТРЕТЬ ВИДЕО» работает, но текстовой ссылки нет.

**Исправление:** заменить `text` на `lesson_text` в вызове `edit_text`, или убрать `lesson_text` и вставить ссылку прямо в `text`.

### 🔴 ОПЕЧАТКА в тексте

`back_to_start` handler, строка 477:
```python
"Для того, чтобы узнать подробнее о курске"  # ← "курске" вместо "курсе"
```

### 🟡 UX: нет прогрева

`/start` → сразу две кнопки «О курсе» / «Купить доступ» без единого слова о том, что это за курс. Холодный старт. (Решается в F1 PRD.)

### 🟡 Email собирается до намерения купить

Текущий порядок: принятие оферты → запрос email → платёж.
Правильный: принятие оферты → платёж... нет, по flow: кнопка купить → оферта → email → платёж.
Email нужно запрашивать последним перед платежом — как есть, но добавить объяснение зачем. (F3 PRD.)

### 🟡 Нет обработки deep link `?start=checklist`

При переходе по `https://t.me/BOT?start=checklist` параметр `checklist` доступен в `message.text` как `/start checklist`, но сейчас не обрабатывается. Нужно для лид-магнита (F2 PRD).

## 7. Лендинг — отсутствует, нужно создать

Текущий маршрут `/` в `main.py` — нужно проверить что там (скорее всего редирект на `/admin` или 404).

### Предлагаемый дизайн лендинга

**Цветовая схема:**
- Фон hero: тёмный градиент `#0f0f1a → #1a1a2e` (premium AI-feel)
- Акцент: фиолетово-синий `#7c3aed` (violet-600) — совпадает с цветом в player.html
- Текст: белый на тёмном (hero), тёмно-серый на светлом (остальное)
- CTA кнопка: `bg-violet-600 hover:bg-violet-700`

**Шрифт:** Inter (уже используется в player.html) — подключить через Google Fonts

**Стек:** Tailwind CDN (уже используется везде) + минимум кастомного CSS

**Ключевые принципы:**
- Один CTA на всю страницу (не 3-4 кнопки)
- Mobile-first (аудитория из Instagram/TikTok)
- Демо-видео autoplay=false, controls visible (пользователь должен нажать сам)
- Нет перегруза — чистые секции с воздухом
